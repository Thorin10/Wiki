Les choix des scans & utilités #top

Nmap : (Système / Réseau)

Flexible : Prend en charge des dizaines de techniques avancées pour mapper des réseaux remplis de filtres IP, pare-feu, routeurs et autres obstacles. Cela inclut de nombreux mécanismes d’analyse de port (TCP et UDP), la détection de système d’exploitation, la détection de version, les balayages de ping, etc. De plus pour avoir Nmap en interface graphic on prend la Zenmap.

Firewalk : (Système)

Est un outil de sécurité réseau de reconnaissance actif qui tente de déterminer les protocoles de couche 4 d'un périphérique de transmission IP donné. Firewalk fonctionne en envoyant des paquets TCP ou UDP avec un TTL supérieur à la passerelle ciblée. Si la passerelle autorise le trafic, elle transmettra les paquets au saut suivant où ils expireront et généreront un message ICMP_TIME_EXCEEDED. Si l'hôte de la passerelle n'autorise pas le trafic, il laissera probablement tomber les paquets sur le sol et nous ne verrons aucune réponse.

Pour obtenir le protocole TTL IP correct qui aboutira à l'expiration des paquets une fois au-delà de la passerelle, nous devons augmenter le nombre de sauts. Nous faisons cela de la même manière que fonctionne traceroute. Une fois que nous avons le compte de passerelle de la passerelle (à ce moment-là, l'analyse est dite `liée`), nous pouvons commencer notre analyse.

Nessus : (Réseau)

Est un outil de sécurité informatique. Il signale les faiblesses potentielles ou avérées sur les machines testées.

Ceci inclut, entre autres :
- les services vulnérables à des attaques permettant la prise de contrôle de la machine, l'accès à des informations sensibles (lecture de fichiers confidentiels par exemple), des dénis de service...
- les fautes de configuration (relais de messagerie ouvert par exemple)
- les patchs de sécurité non appliqués, que les failles corrigées soient exploitables ou non dans la configuration testée
- les mots de passe par défaut, quelques mots de passe communs, et l'absence de mots de passe sur certains comptes systèmes. Nessus peut aussi appeler le programme externe Hydra pour attaquer les mots de passe à l'aide d'un dictionnaire.
- les services jugés faibles (on suggère par exemple de remplacer Telnet par SSH)
- les dénis de service contre la pile TCP/IP

Grabber : (Application)

Est un scanner d'applications Web. Fondamentalement, il détecte certaines vulnérabilités de votre site Web. Grabber est simple, pas rapide mais portable et vraiment adaptable. Ce logiciel est conçu pour numériser de petits sites Web tels que des rencontres personnelles, des forums, etc. Ce n’est absolument pas une application volumineuse : cela prendrait trop de temps et inonderait votre réseau.

Caracteristiques:

• Script intersite
• Injection SQL (il existe également un module spécial Injection SQL aveugle)
• Inclusion de fichier
• Vérification des fichiers de sauvegarde
• Vérification AJAX simple (analyser chaque code JavaScript et obtenir l'URL et essayer d'obtenir les paramètres)
• Analyse hybride / Test de boule de cristal pour une application PHP utilisant PHP-SAT
• Analyseur de code source JavaScript : Evaluation de la qualité / exactitude du JavaScript avec JavaScript Lint
• Génération d'un fichier [session_id, time (t)] pour la prochaine analyse de statistiques

Sqlmap : (Application)

Est un outil de test d'intrusion open source qui automatise le processus de détection et d'exploitation des failles d'injection SQL et de prise en charge des serveurs de base de données. Il est livré avec un puissant moteur de détection, de nombreuses fonctionnalités de niche pour le testeur de pénétration ultime et une large gamme de commutateurs, allant de la prise d'empreinte de base de données à la récupération de données de la base de données, en passant par l'accès au système de fichiers sous-jacent et l'exécution de commandes sur le système d'exploitation. Connexions hors bande

• Prise en charge complète des systèmes de gestion de bases de données MySQL, Oracle, PostgreSQL, Microsoft SQL Server, Microsoft Access, IBM DB2, SQLite, Firebird, Sybase et SAP MaxDB
• Prise en charge complète de six techniques d'injection SQL : aveugle basée sur les booléens, aveugle basée sur le temps, basée sur les erreurs, requête UNION, requêtes empilées et hors bande.
• Prise en charge de la connexion directe à la base de données sans passer par une injection SQL, en fournissant les informations d'identification du SGBD, l'adresse IP, le port et le nom de la base de données.
• Assistance pour énumérer les utilisateurs, les hachages de mots de passe, les privilèges, les rôles, les bases de données, les tables et les colonnes.
• Reconnaissance automatique des formats de hachage de mot de passe et prise en charge de leur résolution à l'aide d'une attaque basée sur un dictionnaire.
• Prise en charge du vidage complet des tables de base de données, d'une plage d'entrées ou de colonnes spécifiques selon le choix de l'utilisateur. L'utilisateur peut également choisir de sauvegarder uniquement une plage de caractères de chaque entrée de colonne.
• Prise en charge de la recherche de noms de bases de données spécifiques, de tables spécifiques dans toutes les bases de données ou de colonnes spécifiques dans toutes les tables de bases de données. Ceci est utile, par exemple, pour identifier les tables contenant les informations d'identification de l'application personnalisée lorsque les noms des colonnes pertinentes contiennent des chaînes telles que name et pass.
• Prise en charge du téléchargement de tout fichier à partir du système de fichiers sous-jacent du serveur de base de données lorsque le logiciel de base de données est MySQL, PostgreSQL ou Microsoft SQL Server.
• Prise en charge de l'exécution de commandes arbitraires et de la récupération de leur sortie standard sur le système d'exploitation sous-jacent du serveur de base de données lorsque le logiciel de base de données est MySQL, PostgreSQL ou Microsoft SQL Server.
• Prise en charge de l’établissement d’une connexion TCP dynamique hors bande entre la machine attaquante et le système d’exploitation sous-jacent du serveur de base de données. Ce canal peut être une invite de commande interactive, une session Meterpreter ou une session VNC (Interface utilisateur graphique) selon le choix de l'utilisateur.
• Prise en charge de l'élévation des privilèges des processus de base de données via la commande Meterpreter getsystem de Metasploit.

Configuration de l’agent Wazuh sur Kali-linux#top

Installer des outils de développement et des compilateurs. Sous Linux, cela peut facilement être fait en utilisant le gestionnaire de paquets de votre distribution :

Pour les distributions basées sur RPM :

                                    $ yum install make gcc policycoreutils-python automake autoconf libtool

Pour les distributions basées sur Debian :

                                    $ apt-get install make gcc libc6-dev curl policycoreutils automake autoconf libtool

Téléchargez et extrayez la dernière version :

                                    $ curl -Ls https://github.com/wazuh/wazuh/archive/v3.8.2.tar.gz | tar zx

Exécutez le install.ssh script. Cela lancera un assistant qui vous guidera tout au long du processus d'installation à l'aide des sources Wazuh : 

                               $ cd wazuh- * && ./install.sh

    • Collecte des logs suite aux scans#top

    On utilise l’agent de Wazuh qui est sur la machine Kali-linux. Nous allons utiliser ici les capacités de collecte et d’analyse de journaux Wazuh pour respecter les contrôles PCI DSS suivants :

  • 10.2.4 : tentatives d'accès logique non valides.
  • 10.2.5 : Utilisation et modification des mécanismes d'identification et d'authentification (y compris, sans toutefois s'y limiter, la création de nouveaux comptes et l'escalade des privilèges), ainsi que de toutes les modifications, ajouts ou suppressions de comptes dotés de privilèges root ou administratifs.

    • Ces contrôles nous obligent à consigner les tentatives d'accès logique non valides, plusieurs tentatives de connexion non valides (attaques par force brute possibles), des hausses de privilèges, des modifications de comptes, etc. Pour cela, nous avons ajouté des balises PCI DSS aux règles d'analyse de journal OSSEC, mappage. Aux exigences correspondantes. Cela facilite l'analyse et la visualisation de nos alertes liées à la norme PCI DSS.

    Nous essayons d'ouvrir le fichier scan.txt.
    Étant donné que notre utilisateur actuel n'a pas d'accès en lecture au fichier, nous sudo exécutons des privilèges élevés. En utilisant le sudo décodeur d’analyse de journal et les règles, Wazuh générera une alerte pour cette action particulière et l’écrira alerts.log. En utilisant les balises de règle, nous pouvons voir quelles exigences PCI DSS sont spécifiquement liées à cette alerte. La sortie JSON étant activée, nous pouvons également voir l'alerte dans alerts.json : Kibana affiche les informations de manière organisée, ce qui permet de filtrer par différents types de champs d'alerte, y compris les contrôles de conformité. Nous avons également développé quelques tableaux de bord PCI DSS pour une visualisation aisée des alertes pertinentes.

    Configuration du VPN #top

    Par défaut, les paramètres réseau n’offrent pas d’option pour établir une connexion VPN. La première étape consiste à ajouter l’option OpenVPN au menu des paramètres réseau. Pour ce faire, nous devons installer OpenVPN afin de pouvoir ajouter une nouvelle connexion VPN dans les paramètres réseau. Commençons l’installation de VPN sur Kali Linux 2016 en utilisant la commande suivante sur une nouvelle session de terminal: 

                                        $ apt-get install network-manager-openvpn-gnome

    Cette commande va installer le gestionnaire de réseau OpenVPN et les paquets supplémentaires qui sont nécessaires pour le faire fonctionner correctement. Le paquet network-manager-openvpn-gnome n’installe que le protocole OpenVPN, c’est suffisant dans la plupart des cas et certainement si vous utilisez le service Golden Frog VPN. Si vous souhaitez installer d’autres protocoles VPN tels que PTTP ou L2TP, vous pouvez utiliser la commande suivante pour les installer tous: 

                                        $ apt-get install network-manager-openvpn network-manager-openvpn-gnome network-manager-pptp
                                    network-manager-pptp-gnome network-manager-strongswan network-manager-vpnc network-managervpnc-gnome

    Lorsque l’installation de tous les paquets est terminée, nous pouvons continuer avec l’étape suivante : Ajouter une nouvelle connexion VPN et configurer les paramètres de connexion VPN. Commençons par ajouter une nouvelle connexion VPN dans les paramètres réseau en accédant au menu des paramètres et ouvrez le panneau des paramètres réseau en utilisant le bouton

    Maintenant, cliquez sur le signe + dans le coin inférieur gauche pour ajouter une nouvelle connexion et cliquez sur “VPN” dans le menu Ajouter une connexion réseau et enfin ajouter une nouvelle connexion OpenVPN

    Cela nous amène au menu où nous pouvons définir les paramètres du serveur VPN pour établir une connexion. Dans cette étape, nous devons définir la passerelle, les informations d’identification de connexion et un certificat d’autorité de certification VyprVPN (CA). Téléchargez d’abord le certificat en utilisant la commande suivante: 

                                        $ wget -O /etc/openvpn/ca.vyprvpn.com.crt
                                    https://support.goldenfrog.com/hc/en-us/article_attachments/205312238/ca.vyprvpn.com.crt

    Assurez-vous que vous enregistrez le fichier de certificat à l’emplacement suivant : /etc/openvpn/
    Maintenant, nous devons spécifier la passerelle et d’autres paramètres :

  • Nom : VyprVPN OpenVPN
  • Rendre disponible aux autres utilisateurs le cas échéant
  • Passerelle : passerelle de serveur VPN, choisissez l’un de ces serveurs de Golden Frog.
  • Nom d’utilisateur : L’adresse e-mail utilisée lors de l’inscription à Golden Frog
  • Mot de passe : le mot de passe de votre compte VyprVPN
  • Certificat CA : sélectionnez le certificat CA téléchargé à partir du répertoire suivant : /etc/openvpn/

    • La dernière étape du processus d’installation consiste à activer la compression LZO dans le menu avancé. Vous pouvez le faire en cliquant sur le bouton avancé dans le coin en bas à droite de l’écran et activer la compression LZO

    Maintenant, fermez le menu des options avancées et appliquez les modifications à la nouvelle connexion VPN et voyons si cela fonctionne en actionnant le commutateur bleu dans le coin supérieur droit de l’écran de connexion VPN pour activer la connexion VPN

    Si tous les paramètres sont valides, vous devriez pouvoir configurer une connexion VPN avec succès. Si des erreurs surviennent, vous serez averti par un message ressemblant à ceci : L’activation de la connexion réseau a échoué. Si ce message d’erreur s’affiche, vous devez vérifier les journaux à l’aide de la commande suivante dans le terminal: journalctl -n100

    Cela ouvrira les 100 dernières lignes du journal. Une erreur dans le journal sera marquée en rouge et devrait éclaircir les raisons pour lesquelles l’erreur est survenue. Dans l’exemple suivant, il y avait un problème avec le certificat téléchargé, qui a été résolu en téléchargeant à nouveau le certificat mais manuellement cette fois en utilisant un navigateur

    Lancement des scripts de scan #top

    On renseigne l'IP de son serveur et il scan tout les ports ouverts sur le serveur, avec les Iptables ont bloquent ceux qui ne servent à rien. Pour le lancer :

  • Python3 : 
    •                                             python3 scan.py
                                        
                                                #!/usr/bin/env python
                                            # coding : utf-8
                                            import socket
                                            import subprocess
                                            import sys
                                            from datetime import datetime

                                            # On efface l'écran
                                            subprocess.call('clear', shell=True)

                                            # On demande à l'utilisateur d'entrer un hôte distant pour pouvoir le scanner
                                            remoteServer    = input("Merci d'entrer l'hôte à scanner: ")
                                            remoteServerIP  = socket.gethostbyname(remoteServer)

                                            # Pour faire patienter notre utilisateur une jolie mise en forme
                                            print ("--" * 60)
                                            print ("Patience scan en cours", remoteServerIP)
                                            print ("--" * 60)

                                            # On effectue une mise à jour de l'heure à laquelle le script a été lancé
                                            t1 = datetime.now()

                                            # Dans une boucle FOR, on utilise RANGE pour définir les ports à scanner (le nombre total des ports est de : 10)
                                            # Il est possible de définir un nombre moins élevé de ports en changeant la valeur 65535 dans le RANGE

                                            # Avec les lignes except.. On vérifie qu'il n'y ait aucune erreur

                                            try:
                                                for port in range(1,10):
                                                    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
                                                    result = sock.connect_ex((remoteServerIP, port))
                                                    if result == 0:
                                                        print ("Port {}:      Open".format(port))
                                                    sock.close()

                                            except KeyboardInterrupt:
                                                print ("Vous avez stoppez le script en appuyant sur Ctrl+C")
                                                sys.exit()

                                            except socket.gaierror:
                                                print ("Le nom de l'hôte n'a pas pu être résolu. Ciao")
                                                sys.exit()

                                            except socket.error:
                                                print ("Impossible de se connecter au serveur")
                                                sys.exit()

                                            # On effectue une deuxième vérification du temps
                                            t2 = datetime.now()

                                            # On calcule grâce à nos deux temps, la différence entre le début et la fin du script
                                            total =  t2 - t1

                                            # On peut maintenant afficher le résultat à l'utilisateur
                                            print ('Scan complet en : ', total)
                                            print ("Port {}:      Open".format(port))
